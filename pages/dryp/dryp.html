JWT authentication

Provides stateless, token-based user verification where the client stores a signed token after login and sends it with each request, enabling lightweight, scalable access control.

Layout sizing

Panels can share or completely occupy the screen real-estate. The navigation menu is also collapsible, in order to increase real-estate.

Map overlays

The Leaflet map supports GeoJSON and image overlays. Sequenced image overlays (like radar images), are cached and playable.

Source virtual

Radar image sequences contain data, that can be derived into time-series, by dragging and placing a virtual source on to the map.

Source sorting

Opened data sources can be dragged to and from different charts. Default sorting can also be set to group by: data-type, location, or no grouping.

Source filtering

Depending on the view, data sources can be filtered in different ways: by grouped locations, sensor type, sensor state, search term, and wether or not they have data.

Source templates

Views and opened data sources can be saved as templates, on a private, organizational, or public level. Handy for lining up usual suspects.

Source SVG icons

Depending on the view, data source icons display either state, type, or both. This is facilitated through dynamic svg elements used in both lists and maps.

Data correlation

Time-series values can be segmented for distribution overview, and grouped time-series can be correlated against eachother in a scatter-plot.

Data annotation

The charts support time-series editing, correction, commenting, and labeling. Theese annotations are represented with a bookmark icon, at the end of the source label.

Annotation inspection

A data source's annotations, can be opened, inspected, and deleted. When opened both edited and raw time-series are displayed on top of eachother.

Source notes

Historical notes are tied to each data source's physical location. A chat layout is used to display the notes, with timestamps, attachments, and author information.

Date/time-range

Time-ranges affect time-series, accumulated data states, sensor states, or the availability of data sources. Defined ranges can be paginated. Some views have default ranges.

Time-series caching

Time-series requests are diffed against cached time blocks to minimize network requests. Missing blocks are coalesced into single requests to reduce round-trips and bandwidth.

Time-series chunking

When there are more data-points than pixels; the time-series are compressed before display for rendering optimization, by dividing into pixel chunks and outputting min/max values per chunk.

Deep linking

Route parameters lets the app encode state directly in the URL, so users can open specific views, data sources, time-ranges, and layouts via links.

Lazy loaded modules

Improves performance by loading feature modules only when needed, reducing initial bundle size and speeding up startup time.

i18n

Internationalization is supported by separating translatable content from code, enabling the app to dynamically load language files and present a fully localized UI.
